# Product Document

## MISSION

Upbit 거래소 API를 활용한 자동 트레이딩 시스템을 통해 개인 투자자가 시장 상황을 24시간 모니터링하고, 데이터 기반의 의사결정으로 자산을 효율적으로 운용할 수 있게 합니다.

## VISION

암호화폐 자동 트레이딩의 접근성을 낮추고, 전문적인 트레이딩 전략을 누구나 쉽게 활용할 수 있는 플랫폼을 제공합니다.

## Problem Statement

### Current Problems (현재 문제점)

1. **시장 상시 모니터링 불가능**
   - 암호화폐 시장은 24시간 운영되지만 개인은 상시 확인 어려움
   - 중요한 가격 변동 시점을 놓치는 경우 빈번

2. **감정 기반 매매**
   - 공포나 탐욕에 의한 충동 매매로 손실 발생
   - 매매 전략 없는 일관성 없는 거래

3. **전략 검증 부족**
   - 과거 데이터로 전략 검증하지 않고 실전 투자
   - 최적 파라미터 찾기 어려움

4. **리스크 관리 부재**
   - 손절가/익절가 설정 없이 보유
   - 포지션 사이즈 조절 실패

### Impact (영향)

- 개인 투자자의 수익률 저하
- 불필요한 손실 발생
- 투자 시간 낭비

## Solution Approach

### Core Solution (핵심 솔루션)

AutoCoin은 **멀티 에이전트 아키텍처**를 기반으로 한 자동 트레이딩 시스템입니다.

### Key Features (주요 기능)

#### 1. 자동 시장 모니터링
- Upbit WebSocket을 통한 실시간 시세 수집
- 상위 20개 코인(시가총액 기준) 동시 모니터링
- 자동 재연결 메커니즘으로 안정성 확보

#### 2. 모멘텀 전략 기반 매수 신호
- 가격 급등 감지 (기본 5% 이상 상승)
- 거래량 급증 분석 (평균의 2배 이상)
- 신호 점수화로 신뢰도 제공

#### 3. 자동 리스크 관리
- 손절가 자동 설정 (기본 -5%)
- 익절가 자동 설정 (기본 +10%)
- 단일 포지션 집중 전략으로 리스크 분산

#### 4. 다양한 전략 지원 (v0.2.0+)
- **모멘텀 전략**: 급등 감지 기반
- **RSI 전략**: 상대 강도 지표 기반
- **MACD 전략**: 이동평균 수렴 발산 기반
- **볼린저 밴드**: 변동성 기반
- **골든크로스**: 이동평균선 교차
- **다중 지표 결합**: 여러 지표를 종합한 신호

#### 5. CLI 대시보드 (v0.2.0+)
- ratatui 기반 TUI 대시보드
- 실시간 에이전트 상태 모니터링
- 포지션 PnL 시각화

#### 6. 웹 대시보드 (v0.3.0+)
- Next.js 16 + React 19 기반
- 실시간 WebSocket 연동
- 반응형 디자인 (데스크톱/모바일)
- 다크 모드 지원
- PnL 차트 및 시장 데이터 시각화
- 수동 매수/매도 기능
- 설정 웹 관리

#### 7. 백테스팅 및 최적화 (v0.2.0+)
- 과거 데이터 기반 전략 검증
- 그리드 서치 기반 파라미터 최적화
- 성과 메트릭 분석 (수익률, 승률, Sharpe Ratio)

#### 8. Discord 알림
- 매수/매도 체결 알림
- 신호 감지 알림
- 에러 발생 알림

## Constraints & Risks (제약 사항 및 리스크)

### Technical Constraints (기술적 제약)

- Upbit API Rate Limit: 초당 10회
- WebSocket 최대 구독: 연결당 300개 채널
- SQLite 동시성 제약

### Business Risks (비즈니스 리스크)

| 리스크 | 영향 | 완화 방안 |
|--------|------|-----------|
| 전략 수익성 부족 | 사용자 이탈 | 백테스팅 기능 강화 |
| API 정책 변경 | 시스템 중단 | 모듈형 아키텍처 |

## Legacy Context (레거시 컨텍스트)

### Existing Issues (기존 이슈)

1. **단일 전략 의존도**: 모멘텀 전략만으로는 시장 상황 대응 부족
   - 해결: 다중 지표 전략 추가 (v0.2.0+)

2. **모니터링 불편**: CLI만으로는 원격 확인 어려움
   - 해결: 웹 대시보드 추가 (v0.3.0+)

3. **전략 검증 부족**: 백테스팅 도구 부족
   - 해결: 백테스팅 시뮬레이터 추가 (v0.2.0+)

### Technical Debt (기술적 부채)

1. **테스트 커버리지**: 현재 약 70%, 목표 85%+
2. **문서화**: API 문서화 완료 필요
3. **에러 처리**: 일부 에러 케이스 처리 개선 필요
